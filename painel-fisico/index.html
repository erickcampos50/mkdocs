Como interpretar os dados que são exibidos no painel físico¶

O painel físico é um monitor conectado a um raspberry pi que fica permanente no local para exibir a todos as informações disponibilizadas pelos sensores. A representação visual dos dados foi pensada para máxima clareza, visto que é algo que será consultado de forma ocasional durante a realização de todas as atividades da rotina do espaço.

Veja abaixo um exemplo das informações que são exibidas no monitor do painel físico.

Vamos agora detalhar a codificação utilizada e como extrair o máximo de informações sobre os qeuipamentos com o meenor tempo de observação.

Lendo os gráficos

Marcação em azul: Indica o setor em que o sensor foi instalado (quando houver um nome para o setor) e um código interno que auxilia na manutenção

Marcação em laranja: Indica quando foi a última vez que o sensor enviou alguma informação. Algo útil para ocasiões em que é preciso realizar o registro das informações.

Marcação em verde: Indica o tipo de sensor (neste caso, trata-se de um sensor de temperatura) e a referência de instalação do sensor (neste caso foi instalado no Freezer 4)

Marcação em vermelho: O valor captado pelo sensor, que neste caso trata-se da temperatura do freezer 4

Seta preta: Indica através da cor de preenchimento se o valor coletado pelo sensor é considerado dentro dos padrões ou não. Essa definição é flexível, podendo ser modificada confroeme a conveniência. A indicação de cor é o principal recurso para alertar aos usuários que algo não está conforme o esperado.

Lendo os gráficos

Para indicar de forma clara e rápida que algum dos sensores econtra-se offline, optou-se por uma indicação bastante chamativa. O texto "Offline" substitui os dados coletados quando não há uma conexão com o sensor.

Isso pode acontecer por causa de problemas na rede, mas também por queda de energia. Por isso quando algum offline for detectado recomanda-se buscar a solução imeditament.
